{% comment %} Shared message-card partial used by home.html, message_search.html, and channel_detail.html so all three render messages identically (images, videos, media placeholders, reactions, replies, details panel). Required context: post — a Message instance. Optional context: show_channel_chip (bool, default False) Render the channel chip at the top. True for home/search pages where the post list spans multiple channels. received_view (bool, default False) Render the "forwarded BY " banner instead of the default "forwarded FROM " — used by channel_detail's "Forwards received" tab where each post lives in a different (forwarding) channel. Views including this partial should prefetch_related("messagepicture_set", "messagevideo_set", "messageaudio_set", "messagesticker_set", "messageothermedia_set", "reactions") to avoid N+1 queries. Telegram media-group albums (multiple photos/videos posted together) are collapsed by `_exclude_album_tails` in webapp/views.py so only the head of each album reaches this template; the head's `album_pictures` / `album_videos` / etc. properties gather media from every sibling message sharing the same (channel_id, grouped_id). {% endcomment %}
